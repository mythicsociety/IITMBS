Introduction: Visualizing Procedures

So far, we've talked about procedures and step-by-step instructions. But how do we write these down so someone else—or even a computer—can understand them clearly? Before we start writing code, it's helpful to have a formal way to map out our logic.

A flowchart is a perfect tool for this. It's a diagram that visually represents the sequence of steps and decisions in a process. By using standard symbols, we can create a clear, language-independent map of our algorithm that anyone can follow.

1. Core Flowchart Symbols

For our purposes, we only need to learn four basic symbols. Mastering these will allow you to map out almost any process we'll encounter.

Terminal

The oval shape indicates the "Start" or "End" of the program.

Process / Activity

A rectangle represents an action, like changing the value of a variable (e.g., `Count = 0`).

Decision

A diamond is used for questions or conditions that determine which path the program takes (e.g., "Are there more cards?").

Flowline

An arrow shows the order of execution, connecting one symbol to the next.
