So far, we've used flowcharts to visualize our algorithms. They are a fantastic tool for understanding the logical flow, decisions, and loops in a process.

However, as our procedures become more complex, drawing and sharing flowcharts can become cumbersome. In this section, we'll introduce a powerful, text-based way to describe algorithms called pseudocode.

Why Not Just Flowcharts?

While flowcharts are great for visualization, they have some practical limitations:

  • Size: Complex algorithms result in large, sprawling flowcharts that are hard to fit on a single screen or page.
  • Collaboration: Sharing and editing a diagram is much harder than sharing and editing a text document. It's difficult for multiple people to work on the same flowchart.
  • Versioning: Tracking changes between different versions of a flowchart is not easy. It's hard to see exactly what was added or modified.

What is Pseudocode?

Pseudocode (which means "fake code") is a bridge between plain English and a real programming language. It's a structured way of writing out an algorithm using text, without worrying about the strict syntax of a specific language like Python or Java.

It uses keywords and indentation to make the structure clear. Key features include:

  • Assignments: Setting the value of a variable, like `count = 0`.
  • Conditional Execution: Performing an action only if a condition is true, using keywords like `if`.
  • Repeated Execution (Loops): Repeating a block of actions as long as a condition is true, using keywords like `while`.

Flowchart vs. Pseudocode: Counting Cards

Let's look at our simple card-counting algorithm in both formats. Notice how the pseudocode captures the exact same logic as the flowchart in a more compact, text-based way.

Flowchart

Pseudocode

count = 0

while (Pile 1 has more cards) {
  Pick a card X from Pile 1
  Move X to Pile 2
  Increment count
}

while (...) { ... } creates a loop. The indented code inside the curly braces `{}` is repeated as long as the condition in the parentheses is true.
