Putting It All Together

We now have all the building blocks to translate our complex flowchart logic into clean, readable pseudocode. This final section will show how to represent iteration with filtering and decision-making using text.

Understanding this translation is key, as pseudocode is the standard way that programmers and computer scientists plan and communicate algorithms before writing actual code.

Core Syntax Review

Let's quickly review the key syntax we've developed:

myValue = 10   // Assignment: sets a variable's value
myValue = myValue + 1   // Update: uses the old value to create a new one

if (X.Gender == "M") {   // Comparison: checks if two values are equal
  // ... do something ...
}

Critical Distinction: `=` vs `==`

A single equals sign (`=`) is for assignment. It means "put the value on the right into the variable on the left". A double equals sign (`==`) is for comparison. It asks the question, "Are these two values the same?" and results in True or False.

Filtering with `if`

Remember our task to sum only the boys' marks? The `if` statement is perfect for this. It acts as a gatekeeper: the update operation only runs if the condition is met.

BoySum = 0

while (Pile 1 has more cards) {
  Pick a card X from Pile 1
  Move X to Pile 2

  if (X.Gender == "M") {
    BoySum = BoySum + X.Maths
  }
}

Branching with `if-else`

When we wanted to sum both boys' and girls' marks, we needed two different actions based on one condition. The `if-else` structure handles this perfectly, representing a fork in the road.

BoySum = 0
GirlSum = 0

while (Pile 1 has more cards) {
  ... pick and move card ...

  if (X.Gender == "M") {
    BoySum = BoySum + X.Maths
  } else {
    GirlSum = GirlSum + X.Maths
  }
}

Finding Max with Pseudocode

Finally, let's translate our "Find Max" flowchart into pseudocode. It uses the same simple `if` structure to conditionally update the `max` variable.

max = 0

while (Pile 1 has more cards) {
  Pick a card X from Pile 1
  Move X to Pile 2

  if (X.Maths > max) {
    max = X.Maths
  }
}
