Introduction: What is a Data Type?

After looking at data sanity, we saw that every piece of data has rules. A student's mark should be a number between 0 and 100, not text. A name shouldn't contain asterisks. You can add two marks together, but you can't add two names.

In programming, we formalize these rules using a concept called a Data Type. A data type tells the computer (and other programmers) two critical things about a piece of data:

  • What are the valid values it can hold? (e.g., numbers from 0 to 100)
  • What are the permitted operations that can be performed on it? (e.g., addition is allowed, but multiplication is not)

By assigning a data type to a variable, we are describing the constraints on that variable, which helps ensure the sanity and correctness of our programs.

1. Basic Data Types

For our datasets, we've encountered three fundamental data types.

Integer

Represents whole numbers, both positive and negative (e.g., ..., -2, -1, 0, 1, 2, ...). Student marks and item quantities are examples of integers.

  • Operations: You can perform arithmetic like addition (+), subtraction (-), multiplication (x), and division (÷).
  • Comparisons: You can also compare integers (e.g., `x > y`). A comparison operation doesn't result in another integer; it results in a Boolean value.

Boolean

The simplest data type. It can only have one of two values: True or False. It's the result of any logical question or comparison.

  • Examples: The result of "Is the student a girl?" is a Boolean. The result of "Are marks > 80?" is a Boolean.
  • Operations: You can combine Boolean values with logical operators like AND and OR.

Character & String

A Character is a single letter, number, or symbol (e.g., 'A', 'z', '7', '$'). A String is a sequence of characters (e.g., "Chennai", "Socks", "It was Monday morning.").

  • Examples: Student names, cities, and the words from our paragraph are all strings.
  • Operations: You can compare strings for equality (e.g., is city == "Chennai"?), but you can't perform mathematical operations like addition on them.

3. Subtypes and Transformation

Sometimes the basic data types are too general. A student's mark is an integer, but it should only be within the range of 0-100. A gender is a character, but it should only be 'M' or 'F'. We can create subtypes to enforce these more specific rules.

What about a date, like "4th June"? It looks like a string, but we want to perform comparisons on it (e.g., is "4th June" before "30th June"?), which is easier with numbers. We can handle this through transformation:

  • We can represent each date as an integer: the day of the year (e.g., Jan 1st = 0, Jan 2nd = 1, ..., Feb 1st = 31).
  • We perform all our logic and comparisons using these integers.
  • When we need to display the date, we transform the integer back into a readable string like "4th June".

This clever trick lets us use the power of integer operations on data that doesn't initially look like a simple number.
