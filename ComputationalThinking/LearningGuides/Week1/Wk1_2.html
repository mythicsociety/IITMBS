Introduction: Thinking Like a Computer

Welcome! This guide will walk you through some of the first big ideas in programming: variables, iteration, and filtering. Before we can tell a computer what to do, we have to learn to organize our thinking in a very specific, systematic way.

Think about following a recipe to cook a meal. The recipe gives you a list of ingredients and a sequence of steps: chop this, heat that, mix them together. Programming is very similar. We give a computer a sequence of steps to achieve a goal. The level of detail in our instructions depends on what the computer (or person) already knows.

You'll find that many problems, even if they look different on the surface (like organizing books on a shelf versus hanging clothes on a line), can be solved using the same general sequence of steps. We call these "patterns," and learning to spot them is the heart of computational thinking.

1. The Concept of a Variable

Imagine you're counting a large pile of cards. If you get interrupted, you might have to start all over again. A better way is to keep track of your progress. You could make a tick mark on a piece of paper for each card, or you could maintain a running total. This "running total" is a variable.

In programming, a variable is a named placeholder that stores a piece of information that can change while a program is running. We give it a name, like count or sum, and we can update its value as we go. For example, we start with count = 0. After the first card, we update it to count = 1, then count = 2, and so on.

Key Idea

A variable is a storage location with a name that holds a value. This value can be updated or changed during a process.

Variables vs. Constants

A variable's value can change. This is different from a constant, which is a value that stays the same. On a student's report card, the student's name (e.g., 'Sofia') or their marks (e.g., Maths: 89) are constants for that specific card. The values we calculate *about* the cards, like the total number of students, are stored in variables because they change as we process each card.

2. Iteration: Repeating Steps Systematically

Let's go back to counting cards. The process of going through each card, one by one, and performing the same action (adding 1 to our count) is called iteration. It's a fundamental pattern for processing collections of data.

An iteration, or a loop, has three key parts:

  • Initialization: Setting things up before you start. For counting, this means creating our count variable and setting it to 0.
  • The Repetitive Step: The action you perform on each item. For counting, it's picking a card, moving it to a "counted" pile, and adding 1 to count.
  • Termination Condition: The rule that tells you when to stop. For counting, we stop when the original pile of cards is empty.

It doesn't matter what order you count the cards in—top to bottom, bottom to top, or randomly—as long as you process each card exactly once, the final count will be correct.

4. Filtering: Selecting Data

Often, we don't want to process every single item in a collection. We only want to work with items that meet a specific condition. This process of selecting a subset of data is called filtering.

For example, from a list of words, we might only want to count the verbs. To do this, we iterate through all the word cards, but we only increment our `verbCount` variable if the "Part of Speech" on the card is 'verb'. If it's a noun or an adjective, we simply ignore it and move to the next card.

This is like an iteration with an "if" condition inside it. For each card, we ask a question (the filter), and only if the answer is "yes" do we perform our action.
